# Plan — #350: validated `AudienceName` newtype (typed wire arg, ADR-0065)

Spec:
[`docs/superpowers/specs/2026-07-15-issue-350-audience-name-newtype.md`](../specs/2026-07-15-issue-350-audience-name-newtype.md).
The spec is the "what/why"; this plan is the "how." Read the spec's **Design**
§1–5 for rationale — not restated here.

## Review header

**Goal.** Encode the "non-empty, trimmed audience name" invariant in a `common`
`AudienceName` newtype, type it as the `create_audience`/`rename_audience`
`#[server]` wire arg, thread it through the store as `&AudienceName`, and
pre-validate client-side (ADR-0065 direct-bind) so legitimate submits never trip
decode-time rejection.

**Scope.**

- _In:_ new `common::audience::AudienceName`; storage signatures →
  `&AudienceName`; web server-fn arg typing + body cleanup; direct-bind client
  validation in the create + rename forms; test updates (common unit, forms host
  test, server integration, e2e).
- _Out:_ read-path re-typing (`AudienceRecord`/`AudienceSummary` stay `String`);
  any charset/length rule beyond trim+non-empty; `<ValidatedInput>` component
  adoption; the forms-abstraction (filed as **#450**, blocked-by this);
  `host/src/error.rs` (no lift needed — typed args never parse in-body).

**Tasks.**

1. `AudienceName` newtype + unit tests in `common`.
2. Thread `&AudienceName` through `AudienceStorage` (trait + impl + binds).
3. Web: typed wire args, body cleanup, direct-bind validation in both forms,
   `forms.rs` host test.
4. Server integration tests: sweep 23 store-seeding call sites to
   `&AudienceName`; rewrite the 2 empty-name tests for decode-time rejection.
5. e2e: add the create-form client-validation test; verify existing audience e2e
   stays green.
6. Full-gate verification.

**Key risks / decisions.**

- **Behavior change (intended):** empty/whitespace `name` now fails at
  **arg-decode** (generic `WebError::ServerFunction`), not the body — so the two
  `*_empty_name_is_validation_error` server tests must drop their
  `body.contains("audience name must not be empty")` assertion (precedent:
  `web_auth.rs::register_invalid_username_returns_error`). The message is
  preserved as the **client-side** inline message.
- **Compilation coupling:** Task 2 breaks the `server` test crate until Task 4's
  sweep lands; each task below still leaves a _different_ target green
  (`-p common` → `-p storage` → `-p web` → `-p server` → e2e), so progress stays
  verifiable.
- **Direct-bind, not `<ValidatedInput>`** — preserves the bespoke markup
  (placeholder, inline rename row) and the existing e2e selectors; sidesteps the
  leptosfmt generic-tag mangling (#420) since there are no generic component
  tags.
- **Ungated import:** `use common::audience::AudienceName;` must be top-level
  (not `#[cfg(feature = "server")]`) — the `#[server]` arg struct references it
  on both builds.

## Global constraints

- **Backend parity:** the storage SQL is shared (single merged impl); both
  backends are covered by the existing dual-backend (`#[apply(backends)]`)
  server tests — no per-dialect change.
- **Coverage:** `AudienceName::from_str` accept/reject is unit-tested in
  `common`; `field_error::<AudienceName>` in the `forms.rs` host test; the
  `#[component]` direct-bind closures are host-compiled dead-but-exempt
  (ADR-0050) and exercised via e2e.
- **Commit:** run `cargo xtask check` before each commit (pre-commit hook runs
  the full gate); **no `Co-Authored-By` trailer** (see `jaunder-commit`).
- **For agentic workers:** drive with `jaunder-iterate`; delegate Task 4's
  mechanical sweep via `jaunder-dispatch`.

```rust
use std::str::FromStr;

use macros::StrNewtype;
use thiserror::Error;

/// A validated audience name: non-empty after trimming, original casing preserved.
///
/// Constructed via [`FromStr`] — the single validating chokepoint; the ADR-0063
/// string-newtype trailer (`Display`, `AsRef`/`Borrow`/`Deref<str>`, owned-`String`
/// conversions, `PartialEq<str>`/`<&str>`, and the validating serde bridge that rejects
/// an empty name on the wire) is generated by `#[derive(StrNewtype)]`. Used as a typed
/// `#[server]` wire arg with client-side pre-validation (ADR-0065). No `Hash`/`Ord`:
/// never a map/set key — DB uniqueness is `UNIQUE (author_user_id, name)`.
#[derive(Clone, Debug, PartialEq, Eq, StrNewtype)]
pub struct AudienceName(String);

/// Error returned when a string cannot be parsed as an [`AudienceName`].
#[derive(Debug, Error)]
#[error("audience name must not be empty")]
pub struct InvalidAudienceName;

impl FromStr for AudienceName {
    type Err = InvalidAudienceName;

    fn from_str(s: &str) -> Result<Self, Self::Err> {
        let trimmed = s.trim();
        if trimmed.is_empty() {
            return Err(InvalidAudienceName);
        }
        Ok(AudienceName(trimmed.to_owned()))
    }
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn parses_and_trims_preserving_case() {
        assert_eq!("  Close Friends ".parse::<AudienceName>().unwrap(), "Close Friends");
        assert_eq!("Family".parse::<AudienceName>().unwrap(), "Family");
    }

    #[test]
    fn rejects_empty_and_whitespace_only() {
        assert!("".parse::<AudienceName>().is_err());
        assert!("   ".parse::<AudienceName>().is_err());
        assert!("\t\n".parse::<AudienceName>().is_err());
    }

    #[test]
    fn display_round_trips_the_name() {
        let n: AudienceName = "Work".parse().unwrap();
        assert_eq!(n.to_string(), "Work");
    }

    #[test]
    fn invalid_error_display_is_the_public_message() {
        assert_eq!(InvalidAudienceName.to_string(), "audience name must not be empty");
    }

    #[test]
    fn serde_serializes_as_plain_string_and_rejects_empty_on_deserialize() {
        let n: AudienceName = "Friends".parse().unwrap();
        assert_eq!(serde_json::to_string(&n).unwrap(), "\"Friends\"");
        assert_eq!(
            serde_json::from_str::<AudienceName>("\" Friends \"").unwrap(),
            "Friends".parse::<AudienceName>().unwrap() // trims on the wire too
        );
        assert!(serde_json::from_str::<AudienceName>("\"   \"").is_err());
    }
}
```

**Step 3.3 — `CreateAudienceForm` direct-bind.** Add a `Field` and wire the
input/submit. Keep the intent comment _outside_ the `view!` macro (leptosfmt
relocates comments near control flow inside `view!`):

```rust
#[component]
fn CreateAudienceForm() -> impl IntoView {
    let create_action = expect_context::<AudienceList>().action::<CreateAudience>();
    // Client-side pre-validation (ADR-0065): the same `AudienceName::from_str` the typed
    // `#[server]` arg decodes through gates submit, so a valid name is a precondition of
    // dispatch and the empty-name rejection never round-trips for a real client.
    let name = Field::<AudienceName>::new();
    view! {
        <section class="j-card">
            <div class="j-card-head">
                <div>
                    <h2>"Create an audience"</h2>
                    <div class="j-sub">
                        "Group your subscribers so you can target posts to a named set."
                    </div>
                </div>
            </div>
            <ActionForm action=create_action>
                <input
                    type="text"
                    name="name"
                    placeholder="Audience name"
                    prop:value=name.value
                    on:input=move |ev| {
                        let v = event_target_value(&ev);
                        name.value.set(v.clone());
                        name.error.set(name.error_for(&v));
                    }
                    on:blur=move |_| name.touch()
                />
                <button
                    type="submit"
                    class="j-btn is-primary"
                    prop:disabled=move || !name.is_valid()
                >
                    "Create"
                </button>
            </ActionForm>
            {move || {
                name.is_touched()
                    .then(|| name.error.get())
                    .flatten()
                    .map(|m| view! { <p class="error">{m}</p> })
            }}
            {move || {
                create_action
                    .value()
                    .get()
                    .and_then(Result::err)
                    .map(|e| view! { <p class="error">{e.to_string()}</p> })
            }}
        </section>
    }
}
```

Note: the `required` attribute is dropped (disable-until-valid replaces it).
