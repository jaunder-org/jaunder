# Milestone 0: Project structure cleanup

## Scope

### Reorganize code

Rename the workspace crates to names that will age well as the project grows.

| Before     | After     | Reason                                                                                                              |
|------------|-----------|---------------------------------------------------------------------------------------------------------------------|
| `app`      | `web`     | "app" collides conceptually with future mobile clients; `web` clearly identifies this as the web interface layer    |
| `frontend` | `hydrate` | `frontend` implies it is the full frontend; in reality it is a thin WASM entry point that only calls `hydrate_body` |
| `server`   | `server`  | Fine as-is                                                                                                          |

### Development process enforcement

Install git hooks to enforce standards on every commit.

## Steps

### Step 1: Rename crates on disk

1. [x] Rename directory `app/` → `web/`
2. [x] Rename directory `frontend/` → `hydrate/`

### Step 2: Update Cargo manifests

1. [x] `web/Cargo.toml`: set `name = "web"`
2. [x] `hydrate/Cargo.toml`: set `name = "hydrate"`; update `app` dependency reference to `web`
3. [x] `server/Cargo.toml`: update `app` dependency reference to `web`
4. [x] Root `Cargo.toml`: update `members` list; update `lib-package` in `[[workspace.metadata.leptos]]` to `"hydrate"`; update `bin-package` to `"server"` (already correct)

### Step 3: Update source references

1. [x] `hydrate/src/lib.rs`: update `use app::*` → `use web::*`
2. [x] `server/src/main.rs`: update `use app::*` → `use web::*`

### Step 4: Verify

1. [ ] `cargo build` succeeds
2. [ ] `cargo nextest run` passes

### Step 5: Install git hooks

Add a `pre-commit` hook that runs the following checks and aborts the commit if any fail:

1. [ ] `cargo fmt --check` — all files are properly formatted
2. [ ] `cargo clippy -- -D warnings` — no lint warnings
3. [ ] `cargo nextest run` — all tests pass
4. [ ] `cargo llvm-cov` with a 100% threshold — coverage is maintained

Add a `pre-push` hook that runs the full end-to-end test suite before pushing:

5. [ ] `nix flake check` — all end-to-end tests pass

The pre-push hook can be skipped for WIP branches by setting `SKIP_E2E=1` before pushing.

All hooks should be committed under `.githooks/`. Document the setup step (`git config core.hooksPath .githooks`) in `CONTRIBUTING.md`.

6. [ ] Create `CONTRIBUTING.md` documenting hook setup

### Step 6: CI enforcement

Configure CI to run all checks as the authoritative gate, blocking merges to `main` if any fail. This backstop ensures standards are enforced even if local hooks are bypassed.

1. [ ] CI runs `cargo fmt --check`, `cargo clippy -- -D warnings`, `cargo nextest run`, on every PR
2. [ ] CI runs `nix flake check` on PRs targeting `main`
